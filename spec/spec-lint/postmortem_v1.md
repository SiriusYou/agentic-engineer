| **正则质量** | 未要求正反例，实现时调试 2 轮 | SDD 中含正反例，实现预期 0 调试 | 改善 |

### 关键改进归因

1. **CLI 专用压测层**（消除 70% 无效题目）
   - TRACK-001 中 U1-U7 对无状态 CLI 全部不适用
   - TRACK-003 中 C1-C5 全部针对 CLI 特征，100% 有效

2. **SDD CLI 变体**（消除接口格式适配摩擦）
   - TRACK-001 需要手动将 HTTP 端点格式改写为命令行参数
   - TRACK-003 直接使用 4.B 变体，exit code 表、stdin/stdout/stderr 契约均有模板

3. **正则正反例必填**（前移正则调试到设计阶段）
   - TRACK-001 的 StepNamingChecker 正则过宽，实现时需 2 轮调试
   - TRACK-003 的所有正则在 SDD 中即有 3 正例 + 3 反例，实现时可直接转为测试用例

4. **首轮即收敛**（压测质量 + 设计精确度双重提升）
   - TRACK-001: 2 轮收敛（Round 1: 1H+4M → Round 2: 0H+0M）
   - TRACK-003: 1 轮收敛（Round 1: 0H+1M → 直接修复通过阈值）
