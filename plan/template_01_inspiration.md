# Template 01 — 灵感整理
# 用途：将原始想法（语音转录 / 代码库探索）整理为结构化需求文本
# 使用对话：任意 AI（Claude / Gemini 均可）
# 新建对话：不需要，可复用

---

## 使用步骤

1. 新建对话
2. 根据输入来源选择对应的 Prompt 模板（A 或 B）
3. 输出结果保存为 `raw_requirements.md`

---

## Prompt A — 语音转录输入

适用于：WisprFlow 等语音录入的零散想法

```
我有一段关于产品/功能的原始想法录音转录，内容比较零散。
请帮我整理成结构化的需求文本，不要添加任何我没有提到的内容，
也不要删减我提到的任何想法，只做结构化整理。

整理格式如下：
1. 核心目标（这个产品/功能要解决什么问题）
2. 目标用户（谁会用它）
3. 核心功能列表（用"用户可以..."句式）
4. 已知约束（技术限制、时间限制、预算限制等）
5. 已知风险（我提到的担忧或不确定点）
6. 明确不做的事（我提到的边界）

原始想法：
[粘贴 WisprFlow 转录文本]
```

## Prompt B — 代码库探索输入

适用于：对已有项目做代码库分析后，想为其新增功能

```
我对一个已有项目做了代码库分析，想为它新增一个功能。
以下是项目概况和功能想法，请整理为结构化需求文本。
不要添加任何我没有提到的内容，也不要删减我提到的任何想法，只做结构化整理。

整理格式如下：
1. 核心目标（这个功能要解决什么问题）
2. 目标用户（谁会用它）
3. 核心功能列表（用"用户可以..."句式）
4. 已知约束（目标项目的技术栈、架构限制等）
5. 已知风险（我提到的担忧或不确定点）
6. 明确不做的事（我提到的边界）

项目概况和功能想法：
[粘贴代码库分析结果和功能想法]
```

---

## 完成标志

输出文本满足以下条件即可进入 Phase 2：
- [ ] 能用一句话说清楚核心目标
- [ ] 核心功能列表有 3 条以上
- [ ] 没有"待定"的关键决策悬而未决

---

## 输出保存

将整理结果保存为：
```
项目目录/
└── spec/
    └── raw_requirements.md
```
