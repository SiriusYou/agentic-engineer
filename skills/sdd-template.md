# SDD 标准模板（软件设计文档）

> 用途：Execution Agent 在读取 spec 时，用此模板验证 spec 是否完整。
> 缺少任意一个 Required 章节，应停止执行并告知用户。

---

## 文档头部

```markdown
# [项目名称] — 软件设计文档 (SDD)
版本: v1.0
状态: Draft / Review / Final
最后更新: YYYY-MM-DD
```

## 4. 接口定义 [Required]

### 4.1 外部 API

每个端点：
```
METHOD /path/to/endpoint
描述: 这个接口做什么

请求:
  Headers: { Authorization: Bearer <token> }
  Body: {
    field_name: type  // 描述，是否必填
  }

响应:
  200: { ... }
  400: { error: string }
  401: { error: "Unauthorized" }

副作用: 会触发什么（发邮件、写数据库等）
```

### 4.2 内部接口/函数签名

```typescript
// 函数名(参数: 类型): 返回类型
// 描述：做什么，什么情况下抛出异常
```

  - (field1, field2): 原因

## 6. 错误处理策略 [Required]

### 错误分类
| 错误类型 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 网络超时 | 重试3次后返回错误 | "请稍后重试" |
| 数据验证失败 | 立即返回400 | 具体字段错误信息 |
| 内部服务错误 | 记录日志，返回500 | "系统错误，请联系支持" |

## Spec 完整性检查

Execution Agent 在开始前验证：
- [ ] 章节 1-6 全部存在
- [ ] 每个组件都有清晰的职责描述
- [ ] 每个 API 端点都有请求/响应格式
- [ ] 所有环境变量都已列出
- [ ] 没有"待定"或"TBD"的关键决策
